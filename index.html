A Unified Diagnostic Framework for GMAT Q, DI, and V Sections

Yuchen Teng

Abstract

  • Unified diagnostic framework for GMAT Q, DI, and V sections
  • Standardized chapter-based methodology beyond accuracy metrics
  • Analyzes time strategy, performance patterns, error causes
  • Identifies root causes for errors and inefficiencies
  • Generates personalized practice recommendations

1. Introduction

  • GMAT requires content mastery and strategic test-taking skills
  • Framework provides in-depth analysis across all scored sections
  • Consistent nine-chapter structure for Q, DI, V sections
  • Motivation: Replace subjective analysis with structured methodology

2.0 Framework Development Methodology

  • Combined quantitative score data with qualitative assessment
  • Formalized diagnostic workflow with precise parameters
  • Iterative validation against expert evaluations
  • Implemented as automated tool with modular design
  • Designed for both student and instructor needs

2.1 Chapter 0: Core Input Data & Configuration

Objective

Define foundational data structures and pre-processing steps

Required Inputs

  • Per-Question: question_id, question_time, is_correct, difficulty, etc.
  • Overall: total_test_time, max_allowed_time, total_questions

Section-Specific Inputs

  • Q: question_type ('Real'/'Pure'), fundamental_skill
  • DI: question_type ('DS', 'TPA', 'MSR', 'GT'), content_domain
  • V: question_type ('CR'/'RC'), fundamental_skill

2.2 Chapter 1: Time Strategy & Data Validity

Objective

Evaluate pacing, time pressure, and identify invalid data points

Key Operations

  • Calculate time difference and pressure status
  • Establish section-specific overtime thresholds
  • Flag invalid data (rushed responses)
  • Create filtered dataset for subsequent analysis
  • Apply overtime rules to filtered data

2.3 Chapter 2: Multi-Dimensional Performance Analysis

Objective

Analyze performance across key dimensions using filtered data

Section-Specific Analysis

  • Q: Compare Real vs Pure, analyze by skill and difficulty
  • DI: Analyze Math vs Non-Math content and question types
  • V: Evaluate performance by skill, difficulty, and question type

2.4 Chapter 3: Root Cause Diagnosis

Time Classification

  • is_relatively_fast: < 75% of average time
  • is_slow: Exceeds overtime threshold
  • is_normal_time: Neither fast nor slow

Special Focus Error (SFE)

  • Wrong answer on question below max mastered difficulty
  • Indicates instability in foundational knowledge

Diagnostic Labels

  • Fast & Right: Fluent Application
  • Slow & Right: Needs Efficiency
  • Fast & Wrong: Potential Carelessness
  • Normal/Slow & Wrong: Knowledge/Process Issues

2.5 Chapters 4 & 5: Efficiency & Pattern Analysis

Key Analyses

  • Correct but overtime questions
  • Early-stage rapid responses
  • Carelessness assessment (fast_wrong_rate > 0.25)
  • DI: MSR time distribution analysis
  • V: CR and RC sub-type breakdown

2.6 Chapter 6: Coverage Rules

Objective

Identify pervasive weaknesses across skills/types

Trigger Conditions

  • Override triggered when error_rate > 0.5 OR overtime_rate > 0.5
  • Sets macroscopic parameters for practice:
    • Y_agg: Minimum difficulty threshold
    • Z_agg: Section-specific time parameters

2.7 Chapter 7: Error Map & Recommendations

Objective

Synthesize findings into structured practice recommendations

Structure

  • Macro-Level: Address foundational weaknesses first
  • Micro-Level: Target specific diagnostic patterns
  • Includes calibrated difficulty and time parameters
  • Prioritizes systemic issues over isolated errors

2.8 Chapter 8: Framework Integration Architecture

Key Components

  • Input handling with validation
  • Sequential chapter-based processing
  • Structured output visualization

Implementation Approaches

  • Manual: Spreadsheets/computational tools
  • Programmatic: Software with modular functions

2.9 Chapter 9: Diagnostic Summary

Key Components

  • Performance overview (metrics and visual representation)
  • Strengths assessment (high-performance areas)
  • Weakness diagnostic (prioritized by coverage rules)
  • Integrated recommendations (practice plan)

Implementation Considerations

  • Technical terminology translated to natural language
  • Personalized to student's performance profile
  • Visual support for key findings
  • Actionable insights directly linked to recommendations
